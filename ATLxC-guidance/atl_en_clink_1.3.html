EXEC CCALL

  1. Function

    This utility calls the C program functions from the ATL program.

  2. Syntax 

    

    EXEC CCALL(FUNC,ARG1,ARG2,...)

    • FUNC

      Specify the function name to be called.

      Specify the function name by using the STRING variable or a character constant.

      This argument cannot be omitted.

      CAUTION

      If an MCI executed by the function generates a runtime error, the test plan program execution terminates.

      • Branch processing is not performed by the ON or ONCALL statement.

      • The setting for error break specified by using Lib Debug is ignored.

    • ARG1,ARG2,...

      Specify the arguments that are passed to the function to be called. An arbitrary number and type of variables can be specified.

      However, the arguments must be specified by using variables when executing CCALL.

      These arguments can be omitted.

      Up to 35 arguments can be specified.

      In CCALL, the memory addresses that correspond to the arguments (ARG1, ARG2,...) are regarded as separate addresses and are passed to the called function that is specified in FUNC.

      The arguments that are passed to CCALL must be specified by using variables.

      The function specified in FUNC must receive as many arguments as the arguments that were passed to CCALL and from which FUNC is excluded.

      However, CCALL does not concern the number of arguments that are received by the called function.

      The application that uses CCALL matches the number of arguments passed and the number of arguments received.

      • ATL program 

        

EXEC CCALL("FUNC",ARG1,ARG2,...ARGn)

      • C program 

        

void FUNC(void *arg1 , void *arg2 , ... ,void *argn)

      CAUTION

      When reading or writing variable areas, use the dedicated API functions.

      The variable areas must not be read or written directly.

      For more information on the API functions, refer to the following:

      Variable Area READ/WRITE API Functions
